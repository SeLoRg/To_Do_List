1. Сессия - объект, которые управляет взаимодействием с базой днанных
Она использует механизм кеширование, который хранит уникальные экземляры объектов
Она представляет методы для выполения запросов к базе данных

2. Движок базы данных - нужен для выполенния операций  с базой данных без блокировки основного потока. Создается на основе url

3. Фабрика сессий - это фабрика создания сессий для управления взаимодействием с базой данных

4.async_scoped_session: Эта функция создает управляемую сессию, которая использует указанную фабрику (self.session_factory) для создания новых экземпляров AsyncSession. Она также связывает каждую сессию с текущей задачей (через scopefunc=current_task), что позволяет каждой асинхронной задаче иметь свою собственную изолированную сессию.

5.Depends - управляет зависимостями в пиложении
